# Frontend Design Skill

## Required References

- Project design system.
- Project context.
- Existing app shell and shared styles.

## Principles

- Design for the product's domain, not generic SaaS sameness.
- Use design tokens and shared classes.
- Prefer dense operational layouts for enterprise tools.
- Keep interaction states obvious and accessible.
- Use real data shapes and expected workflows.

## Must Do

- Support light and dark themes if the project supports them.
- Add ARIA labels to icon-only or ambiguous controls.
- Ensure text fits on mobile and desktop.
- Use stable dimensions for fixed-format controls.
- Verify with type-check/build and browser inspection.

## Avoid

- Decorative blobs/orbs as filler.
- Hardcoded theme colors.
- Nested card-heavy layouts.
- Landing pages when the user asked for an app/tool.
- Visible explanatory text that merely describes UI features.
