# Repository Hygiene Archive - 2026-06-13

This folder contains files moved out of the active source tree during issue #273.
They were archived instead of deleted so historical context remains available
without implying that the files are active platform inputs.

## Archived Groups

- `planning/` - superseded or unreferenced planning notes that are not part of the
  current runtime, deployment, SDLC, or product context loading path.
- `docs-team/` - unreferenced attachment-style documentation artifacts.
- `diagnostics/` - one-off local API and database probes with hardcoded local URLs
  or demo credentials. These are not imported by the API, tests, CI, Docker, or
  Hostinger deployment path.
- `runtime/` - tracked PID files generated by local server runs. Regenerated PID
  files are now ignored by `.gitignore`.

## Files Intentionally Left Active

- `docs/team/CODEX_CONTEXT.md`, `docs/team/SDLC_PROTOCOL.md`, Hostinger runbooks,
  release evidence, and security/QA evidence remain active because project
  instructions or operational workflows reference them.
- `team/DESIGN_SYSTEM.md` remains active because frontend instructions and the
  design skill still name it as the source of truth.
- `productupgrade.md` and `productupgrade_addendum.md` remain active because
  agent instructions and the configurable-platform ledger still reference them.
